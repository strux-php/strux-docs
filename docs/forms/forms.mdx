# Forms

The forms system provides a structured, attribute-driven way to build, validate, and render HTML forms. It follows a **declarative pattern**: you define what fields you want using PHP attributes, and the framework handles the rest — rendering HTML, binding request data, validating input, and surfacing errors.

---

## Architecture Overview

The form system has three layers:

1. **Declaration** — You define a form class with typed properties annotated with field attributes (e.g. `#[StringField]`, `#[EmailField]`).
2. **Binding & Validation** — The form binds to a data source (HTTP request, array, or model object), validates the data against rules, and populates field values.
3. **Rendering** — In your template, you render the form's fields as HTML using methods like `->input()`, `->label()`, and `->error()`.

```
                   ┌─────────────────┐
                   │   ProductForm   │
                   │  (your class)   │
                   └────────┬────────┘
                            │ extends
                   ┌────────▼────────┐
                   │      Form       │
                   │  (abstract base)│
                   └────────┬────────┘
                            │ uses
             ┌──────────────┼──────────────┐
             │              │              │
      ┌──────▼──────┐ ┌────▼─────┐ ┌──────▼──────┐
      │ parseAttrs  │ │  bind()  │ │  isValid()  │
      │ → discover  │ │ → import │ │ → validate  │
      │   fields    │ │   data   │ │   fields    │
      └─────────────┘ └──────────┘ └─────────────┘
```

## Creating a Form

Every form class must extend `Form` (which implements `FormInterface`). Each field is a **protected typed property** annotated with a field attribute:

```php
use Strux\Component\Form\Form;
use Strux\Component\Form\Attributes\StringField;
use Strux\Component\Form\Attributes\EmailField;
use Strux\Component\Form\Attributes\TextAreaField;
use Strux\Component\Form\Attributes\SubmitField;

class ContactForm extends Form
{
    #[StringField(label: 'Your Name', rules: ['required', 'minLength:3'])]
    protected string $name;

    #[EmailField(label: 'Email Address', rules: ['required', 'email'])]
    protected string $email;

    #[TextAreaField(label: 'Message', rules: ['required', 'minLength:10'])]
    protected string $message;

    #[SubmitField(label: 'Send Message')]
    protected string $submit;
}
```

### The `build()` Method

Some fields cannot be declared purely as attributes because they need dynamic configuration (e.g. options loaded from the database). Use the `build()` method for these:

```php
use Strux\Component\Form\Attributes\SelectField;
use Strux\Component\Form\Attributes\ListField;

class ProductForm extends Form
{
    #[SelectField(label: 'Brand', options: [])]   // empty — populated in build()
    protected string $brandId;

    public function build(): void
    {
        $brands = Brand::query()->orderBy('name')->get();
        $options = ['' => 'No Brand'];
        foreach ($brands as $b) {
            $options[(string) $b->brandId] = $b->name;
        }
        $this->field('brandId')->setOptions($options);
    }
}
```

`build()` is called automatically during construction, after the attributes are parsed and before data binding.

## Field Types Reference

Every field attribute maps to an HTML input type and optionally applies coercion (automatic type casting of the submitted value).

### Text & String Inputs

| Attribute | HTML `<input type>` | Coercion | Description |
|---|---|---|---|
| `StringField` | `text` | — | Single-line text input |
| `TextAreaField` | `<textarea>` | — | Multi-line text input |
| `EmailField` | `email` | — | Email input (triggers email keyboard on mobile) |
| `PasswordField` | `password` | — | Password input (value NOT retained on form errors) |
| `HiddenField` | `hidden` | — | Hidden field, no label |
| `SearchField` | `search` | — | Search input with clear button |
| `URLField` | `url` | — | URL input (triggers URL keyboard) |
| `TelField` | `tel` | — | Telephone input (triggers dialpad) |

```php
#[StringField(label: 'Full Name', rules: ['required'])]
protected string $name;

#[TextAreaField(label: 'Bio', attributes: ['rows' => 5, 'placeholder' => 'Tell us about yourself...'])]
protected string $bio;

#[EmailField(label: 'Email', rules: ['required', 'email'])]
protected string $email;

#[PasswordField(label: 'Password', rules: ['required', 'minLength:8'])]
protected string $password;

#[HiddenField()]
protected string $token;

#[SearchField(label: 'Search')]
protected string $query;

#[URLField(label: 'Website')]
protected string $website;

#[TelField(label: 'Phone')]
protected string $phone;
```

### Numeric Inputs

| Attribute | HTML `<input type>` | Coercion | HTML attributes |
|---|---|---|---|
| `IntegerField` | `number` | `int` | `step="1"`, `inputmode="numeric"` |
| `DecimalField` | `text` | `float` | `step="0.01"`, `inputmode="decimal"` |
| `FloatField` | `number` | `float` | `step="any"`, `inputmode="decimal"` |

```php
#[IntegerField(label: 'Quantity', rules: ['required', 'integer', 'min:1', 'max:100'])]
protected string $quantity;

#[DecimalField(label: 'Price', rules: ['required', 'numeric'])]
protected string $price;

#[FloatField(label: 'Rating')]
protected string $rating;
```

**Important:** `DecimalField` renders as `type="text"` (not `type="number"`) because `type="number"` with decimal step can be unpredictable across browsers. The field is functionally numeric via the `numeric` validation rule and coerced to `float` on the server side.

### Boolean Inputs

| Attribute | HTML | Coercion | Behavior |
|---|---|---|---|
| `BooleanField` | `<input type="hidden">` + `<input type="checkbox">` | `bool` | Submits `0` when unchecked, `1` when checked |

```php
#[BooleanField(label: 'Subscribe to newsletter')]
protected string $subscribe;
```

The dual-input trick (hidden + checkbox) ensures a value is always submitted — `0` when unchecked, `1` when checked. Without the hidden input, unchecked checkboxes send nothing at all.

**How checked is determined:**
```php
$checked = $value === true || $value === 1 || $value === '1' || $value === 'on';
```

#### `ListField` — Checkbox List (sub-field)

`ListField` renders as a list of checkboxes. It accepts a `fieldType` parameter that specifies which field attribute the individual checkboxes should behave like (defaults to `BooleanField`):

```php
use Strux\Component\Form\Attributes\ListField;

#[ListField(
    label: 'Categories',
    options: ['1' => 'Electronics', '2' => 'Clothing', '3' => 'Books'],
    fieldType: BooleanField::class,
    multiple: true,
)]
protected string $categories;
```

The `toFieldConfig()` method on `ListField` passes `fieldType` and `multiple` to the `Field` object, which uses them in `renderList()`:

```php
public function toFieldConfig(): array
{
    return array_merge(parent::toFieldConfig(), [
        'fieldType' => $this->fieldType,
        'multiple' => $this->multiple,
    ]);
}
```

| Attribute | HTML | Coercion | Notes |
|---|---|---|---|

| Attribute | HTML | Notes |
|---|---|---|
| `SubmitField` | `<input type="submit">` | Default class `btn btn-primary` |

`SubmitField` renders as `<input type="submit">`, while `ButtonField` renders as `<button type="submit">`. Both serve the same purpose but use different HTML elements. Use `SubmitField` for simple forms and `ButtonField` when you need to nest content inside the button (icons, etc.).

## Field Configuration Reference

Every field attribute accepts a common set of parameters via the parent `FieldAttribute` class:

```php
public function __construct(
    public string  $type = 'text',       // HTML input type
    public ?string $label = null,        // Display label (null = no label)
    public array   $rules = [],          // Validation rules
    public array   $attributes = [],     // Custom HTML attributes
    public array   $options = [],        // For select/radio/checkbox options
    public mixed   $default = null,      // Default value
    public ?string $coerce = null        // Type coercion (int, float, bool, string, array)
)
```

### The `fromConfig()` Method

On the attribute class:

```php
public static function fromConfig(array $config): static
{
    $ref = new ReflectionClass(static::class);
    $ctor = $ref->getConstructor();
    $params = $ctor ? $ctor->getParameters() : [];

    $args = [];
    foreach ($params as $param) {
        $name = $param->getName();
        if (array_key_exists($name, $config)) {
            $args[$name] = $config[$name];
        } elseif ($param->isDefaultValueAvailable()) {
            $args[$name] = $param->getDefaultValue();
        }
    }

    return new static(...$args);
}
```

This is used during `Form::add()` when a field type is a subclass of `FieldAttribute`. It allows creating attribute instances from a plain config array, enabling the dual declaration path (attribute vs `build()` method).

## Form Binding (Data Flow)

Binding is the process of populating form fields with data. The form's `bind()` method accepts three types of data:

### Binding an HTTP Request

```php
// In your controller:
$form = new ContactForm();

if ($request->is('POST')) {
    $form->bind($request);  // Extracts all POST data via $request->all()

    if ($form->isValid()) {
        // Process the form...
        $name = $form->get('name');
        $email = $form->get('email');
    }
}
```

The `bind()` method detects the type of data automatically:

```php
public function bind(mixed $data): self
{
    $inputData = [];

    if ($data instanceof Request) {
        $inputData = $data->all();          // HTTP request → all input
    } elseif (is_array($data)) {
        $inputData = $data;                 // Plain array → direct use
    } elseif (is_object($data)) {
        // Model/entity → extract matching properties
        foreach ($this->fields as $name => $field) {
            if (property_exists($data, $name)) {
                $value = $data->$name;
                if (is_scalar($value)) {
                    $inputData[$name] = (string) $value;
                } elseif (is_array($value)) {
                    $inputData[$name] = $value;
                } elseif ($value instanceof \DateTime) {
                    $inputData[$name] = $value->format('Y-m-d\TH:i:s');
                }
            } elseif (method_exists($data, 'get' . ucfirst($name))) {
                $method = 'get' . ucfirst($name);
                $value = $data->$method();
                if (is_scalar($value)) {
                    $inputData[$name] = (string) $value;
                } elseif (is_array($value)) {
                    $inputData[$name] = $value;
                }
            } elseif (method_exists($data, 'toArray')) {
                $arrayData = $data->toArray();
                if (isset($arrayData[$name])) {
                    $inputData[$name] = (string) $arrayData[$name];
                }
            }
        }
    }
    // ... then sets field values and properties
}
```

### Binding an Entity/Model

```php
$product = $this->productRepository->findById($id);
$form = new ProductForm($product);  // Auto-binds via constructor
```

When the form receives an object, it looks for each field name as:
1. **A public property** on the object (e.g. `$product->title`)
2. **A getter method** (e.g. `$product->getTitle()`)
3. **A `toArray()` fallback** on the object

### Auto-Binding via Constructor

If you pass data to the constructor, `bind()` is called automatically:

```php
public function __construct(mixed $data = null, ?ValidatorInterface $validator = null)
{
    // 1. Resolve validator
    // 2. Parse attributes + call build()
    // 3. Auto-bind:
    if ($data !== null) {
        $this->bind($data);
    }
}
```

### Smart Type Handling

When binding to an object, the form system handles different types:

| Object Property Type | Converted To | Example |
|---|---|---|
| `string`, `int`, `float`, `bool` | `(string) $value` | `"Hello"` |
| `array` | Kept as array | `['red', 'blue']` |
| `DateTime` | `$value->format('Y-m-d\TH:i:s')` | `"2026-05-15T19:52:00"` |
| `null` | Not included (field stays at default) | — |

### Re-binding After Validation Failure

```php
$form->bind($request);  // Bind user input

if (!$form->isValid()) {
    // Form is re-rendered with submitted values + error messages
    return $this->view('contact', ['form' => $form]);
}
```

After `bind()`, the form's field values contain the submitted data, so re-rendering the template automatically shows what the user typed.

## Accessing Form Data

### `get()`

Get a single field's value:

```php
$title = $form->get('title');          // Returns the value or null
$title = $form->get('title', 'Untitled');  // With default fallback
```

### `getData()`

Get all field values as an associative array:

```php
$data = $form->getData();
// ['title' => 'My Product', 'price' => '19.99', ...]
```

### `field()->getValue()`

Get the raw value from the underlying `Field` object (bypasses any form-level processing):

```php
$value = $form->field('categories')->getValue();
```

### Magic Property Access

Form fields are accessible as magic properties via `__get()`:

```php
<?= $form->email->label() ?>    <!-- Same as $form->field('email')->label() -->
<?= $form->email->input() ?>    <!-- Same as $form->field('email')->input() -->
<?= $form->email->error() ?>    <!-- Same as $form->field('email')->error() -->
```

This works because `Form::__get()` checks `$this->fields[$name]` before falling back to class properties:

```php
public function __get(string $name)
{
    if (isset($this->fields[$name])) {
        return $this->fields[$name];
    }
    if (property_exists($this, $name)) {
        return $this->$name;
    }
    throw new \InvalidArgumentException("Field or property '$name' does not exist.");
}
```

Both forms work identically:

```php
// Long form (explicit):
<?= $form->field('email')->label() ?>
<?= $form->field('email')->input(['class' => 'form-input']) ?>
<?= $form->field('email')->error('text-red-500') ?>

// Magic property (shorthand):
<?= $form->email->label() ?>
<?= $form->email->input(['class' => 'form-input']) ?>
<?= $form->email->error('text-red-500') ?>
```

**Important:** The magic property access works even when the form property is `protected`. `__get()` is triggered for inaccessible properties, so `$form->email` returns the `Field` object regardless of the `$email` property's visibility.

## Validation

### Defining Rules

Rules are defined as strings in the field attribute's `rules` parameter:

```php
#[StringField(label: 'Username', rules: ['required', 'minLength:3', 'maxLength:50', 'alpha'])]
protected string $username;
```

### Rule Syntax

Each rule string is parsed using this pattern:

```php
// Simple rule (no parameters):
'required'

// Rule with parameters (colon-separated value, comma-separated for multiple params):
'minLength:3'
'maxLength:100'
'in:admin,user,editor'

// Rule with custom error message (message after space):
'required:Please fill this field'
'minLength:3:Must be at least 3 chars'
```

The rule resolution in `Validator::resolveRule()`:

```php
private function resolveRule(string $ruleString): RulesInterface
{
    $params = [];
    if (str_contains($ruleString, ':')) {
        [$ruleName, $paramStr] = explode(':', $ruleString, 2);
        $params = explode(',', $paramStr);
    } else {
        $ruleName = $ruleString;
    }

    $className = 'Strux\\Component\\Validation\\Rules\\' . ucfirst(strtolower($ruleName));

    if (class_exists($className)) {
        return new $className(...$params);
    }

    throw new \RuntimeException("Validation rule class '{$className}' not found.");
}
```

**Important note on custom messages:** Rules accept custom messages via their constructor. When using string syntax, the first parameter after the colon is the rule value (not the message), and the message cannot currently be customized via string syntax. Custom messages require using the `Callback` rule or creating a custom rule class.

### Available Rules (21 rules)

All rules implement `RulesInterface`:

```php
interface RulesInterface
{
    public function validate($value, mixed $data = null): ?string;
    // Returns null if valid, error string if invalid
}
```

#### Presence & Structure

| Rule | Parameters | Validates | Example |
|---|---|---|---|
| `required` | — | Value is not empty (`!empty($value)`) | `rules: ['required']` |
| `alpha` | — | Contains only alphabetic characters (`ctype_alpha`) | `rules: ['alpha']` |
| `alphanumeric` | — | Contains only letters and numbers (`ctype_alnum`) | `rules: ['alphanumeric']` |

#### Length

| Rule | Parameters | Validates | Example |
|---|---|---|---|
| `minLength` | `N` (int) | Length >= N (`strlen`) | `rules: ['minLength:3']` |
| `maxLength` | `N` (int) | Length &lt;= N (`strlen`) | `rules: ['maxLength:100']` |

#### Numeric

| Rule | Parameters | Validates | Example |
|---|---|---|---|
| `numeric` | — | Is numeric (`is_numeric`) | `rules: ['numeric']` |
| `integer` | — | Is integer (`filter_var FILTER_VALIDATE_INT`) | `rules: ['integer']` |
| `min` | `N` (float) | Value >= N | `rules: ['min:1']` |
| `max` | `N` (float) | Value &lt;= N | `rules: ['max:1000']` |

#### Format

| Rule | Parameters | Validates | Example |
|---|---|---|---|
| `email` | — | Valid email (`filter_var FILTER_VALIDATE_EMAIL`) | `rules: ['email']` |
| `url` | — | Valid URL (`filter_var FILTER_VALIDATE_URL`) | `rules: ['url']` |
| `date` | — | Valid date string (`strtotime !== false`) | `rules: ['date']` |
| `time` | — | HH:MM format (`preg_match /^([01]\d\|2[0-3]):([0-5]\d)$/`) | `rules: ['time']` |
| `regex` | Pattern | Matches pattern (`preg_match`) | `rules: ['regex:/^[A-Z]+$/']` |

#### Comparison

| Rule | Parameters | Validates | Example |
|---|---|---|---|
| `equal` | `field_name` | Value equals another field's value | `rules: ['equal:password_confirmation']` |
| `same` | `field_name` | Value is identical to another field's value | `rules: ['same:emailConfirmation']` |
| `in` | `val1,val2,...` | Value is in allowed list | `rules: ['in:active,inactive,draft']` |
| `notin` | `val1,val2,...` | Value is NOT in disallowed list | `rules: ['notin:admin,superadmin']` |

**Difference between `equal` and `same`:**
- `equal`: Returns a default message `"Field do not match {otherField}"` when values differ
- `same`: Returns a custom message `"The field must match {otherField}"` when values differ
- Both check `$value !== $data[$this->otherField]`
- Both require the other field to exist in the data array

#### Security

| Rule | Parameters | Validates | Example |
|---|---|---|---|
| `passwordStrength` | — | At least 1 uppercase, 1 lowercase, 1 digit, 1 special char, 8+ chars | `rules: ['passwordStrength']` |

#### Custom

| Rule | Parameters | Validates | Example |
|---|---|---|---|
| `callback` | Closure | Custom validation logic | See example below |

#### Callback Rule (Custom Validation)

The callback rule lets you write inline validation logic:

```php
use Strux\Component\Validation\Rules\Callback;

#[StringField(label: 'Coupon Code', rules: [
    'required',
    new Callback(function ($value, $data) {
        // Return true = valid
        // Return string = error message
        if (!str_starts_with($value, 'SAVE')) {
            return 'Coupon code must start with "SAVE".';
        }
        return true;
    }),
])]
protected string $couponCode;
```

The `Callback` rule class:

```php
class Callback implements RulesInterface
{
    public function __construct(callable $callback, ?string $message = null)
    {
        $this->callback = $callback instanceof Closure ? $callback : $callback(...);
        $this->message = $message ?? 'Custom validation failed.';
    }

    public function validate($value, $data = null): ?string
    {
        $result = ($this->callback)($value, $data);

        if ($result === true) {
            return null;
        }

        return is_string($result) ? $result : $this->message;
    }
}
```

### Validation Lifecycle

```
$form = new ContactForm();          // Fields created, no data
$form->bind($request);              // Data bound, isBound = true
$form->isValid()                    // Triggers validation
    ├── Gathers field values and rules
    ├── Passes to $validator->validate($data, $rules)
    │       └── For each field:
    │           ├── resolveRule('required') → new Required()
    │           ├── resolveRule('email') → new Email()
    │           ├── $rule->validate($value, $data) → null or error string
    │           └── Stores errors by field name
    ├── If invalid: sets errors on corresponding Field objects
    └── Returns true/false
```

### Error Messages

The `isValid()` method populates errors on individual fields:

```php
public function isValid(): bool
{
    if (!$this->isBound) return false;

    $data = [];
    $rules = [];

    foreach ($this->fields as $name => $field) {
        $data[$name] = $field->getValue();
        $fieldRules = $field->getRules();
        if (!empty($fieldRules)) {
            $rules[$name] = $fieldRules;
        }
    }

    if ($this->validator->validate($data, $rules)) {
        return true;
    }

    $errors = $this->validator->getErrors();
    foreach ($errors as $field => $messages) {
        if (isset($this->fields[$field])) {
            $this->fields[$field]->setErrors((array) $messages);
        }
    }

    return false;
}
```

Individual validation rules return `null` (valid) or `string` (error message). The first rule failure for each field short-circuits further validation (the `break` in `validateField()`).

## Rendering Forms

### In the Controller

Pass the form to your view:

```php
public function create(): Response
{
    $form = new ContactForm();
    return $this->view('contact', ['form' => $form]);
}
```

### In the Template (Plates)

```php
<form method="POST" action="/contact">
    <div>
        <?= $form->field('name')->label() ?>
        <?= $form->field('name')->input(['class' => 'form-control']) ?>
        <?= $form->field('name')->error('text-red-500 text-sm') ?>
    </div>

    <div>
        <?= $form->email->label() ?>
        <?= $form->email->input(['class' => 'form-control']) ?>
        <?= $form->email->error('text-red-500 text-sm') ?>
    </div>

    <div>
        <?= $form->field('message')->label() ?>
        <?= $form->field('message')->input(['class' => 'form-control', 'rows' => 5]) ?>
```

### Rendering Methods on `Field`

#### `label(array $attributes = []): string`

Renders the `<label>` element. The `for` attribute is automatically set to the field name. No label is rendered for `submit`, `button`, `reset`, or `hidden` field types.

```php
<?= $form->field('email')->label() ?>
<!-- <label for="email">Email Address</label> -->

<?= $form->field('email')->label(['class' => 'font-bold']) ?>
<!-- <label for="email" class="font-bold">Email Address</label> -->
```

#### `input(array $attributes = []): string`

Renders the full HTML input, select, textarea, or checkbox group. The attributes passed here **merge with** the attributes defined in the field attribute declaration (with passed attributes taking priority).

```php
<?= $form->field('email')->input() ?>
<!-- <input type="email" name="email" value=""> -->

<?= $form->field('email')->input(['class' => 'form-input', 'placeholder' => 'you@example.com']) ?>
<!-- <input type="email" name="email" value="" class="form-input" placeholder="you@example.com"> -->
```

If the field has validation errors, the `is-invalid` CSS class is appended automatically:

```php
// If field has errors:
<?= $form->field('email')->input(['class' => 'form-input']) ?>
<!-- <input type="email" ... class="form-input is-invalid"> -->
```

#### `error(string $class = 'invalid-feedback'): string`

Renders the first validation error message inside a `<div>`. Returns empty string if the field has no errors.

```php
<?= $form->field('email')->error() ?>
<!-- <div class="invalid-feedback">Please enter a valid email address.</div> -->

<?= $form->field('email')->error('text-red-500 text-xs mt-1') ?>
<!-- <div class="text-red-500 text-xs mt-1">Please enter a valid email address.</div> -->
```

### Field Type Rendering Details

Each field type is rendered by a dedicated method on `Field`:

#### Checkbox (`BooleanField`)

Renders a hidden input + checkbox to ensure a `0` value is always submitted:

```php
// $checked = true
<input type="hidden" name="subscribe" value="0">
<input type="checkbox" name="subscribe" value="1" checked>

// $checked = false
<input type="hidden" name="subscribe" value="0">
<input type="checkbox" name="subscribe" value="1">
```

#### Radio (`RadioField`)

Renders multiple `<label><input type="radio"> <span>...</span></label>` elements, each with a unique `id` based on the value:

```php
<label class="radio-option">
    <input type="radio" name="gender" value="male" id="gender_male" checked>
    <span>Male</span>
</label>
<label class="radio-option">
    <input type="radio" name="gender" value="female" id="gender_female">
    <span>Female</span>
</label>
```

#### Select (`SelectField` / `SelectMultipleField`)

Single select:
```php
<select name="country">
    <option value="">Select...</option>
    <option value="za" selected>South Africa</option>
    <option value="ng">Nigeria</option>
</select>
```

Multiple select (name ends with `[]`):
```php
<select name="permissions[]" multiple>
    <option value="read" selected>Read</option>
    <option value="write">Write</option>
</select>
```

#### Textarea (`TextAreaField`)

```php
<textarea name="bio" rows="5" class="form-input">Existing value</textarea>
```

#### Checkbox List (`ListField`)

Renders a hidden input + individual checkboxes:

```php
<input type="hidden" name="categories[]" value="">
<label class="checkbox-option">
    <input type="checkbox" name="categories[]" value="1" checked> <span>Electronics</span>
</label>
<label class="checkbox-option">
    <input type="checkbox" name="categories[]" value="2"> <span>Clothing</span>
</label>
```

### Custom Layout Callback

For full control over field rendering, pass a callback as the second argument. The callback receives all fields and the form instance:

```php
<?= $form->render(['class' => 'grid grid-cols-1 md:grid-cols-2 gap-6'], function($fields, $form) {
    $html = '';
    foreach ($fields as $field) {
        $type = $field->getType();
        $spanClass = '';

        // Span full-width for textareas, checkboxes, and submit buttons
        if (in_array($type, ['textarea', 'checkbox', 'submit'])) {
            $spanClass = 'md:col-span-2';
        }

        $html .= sprintf('<div class="form-field %s">', $spanClass);
        $html .= $field->label(['class' => 'form-label']);
        $html .= $field->input(['class' => 'form-input']);
        $html .= $field->error('form-error');
        $html .= '</div>';
    }
    return $html;

```

**Why use a layout callback?**
- Control grid spanning per field type
- Add custom wrappers or icons around specific fields
- Group related fields together
- Apply different CSS classes per field

The `add()` method is **chainable**:

```php

     ->add('dueDate', DateField::class)
```

The `add()` method resolves the field type:

```php
protected function add(string $name, string $type, array $config = []): self
{
    $fieldType = $type;
    $fieldConfig = $config;

    if (is_subclass_of($type, FieldAttribute::class)) {
        // Create attribute instance from config, extract type and config
        $attribute = $type::fromConfig($config);
        $fieldType = $attribute->type;
        $fieldConfig = array_merge($attribute->toFieldConfig(), $config);
    }

    $this->fields[$name] = new Field($name, $fieldType, $fieldConfig);
    return $this;
}
```

## The `FieldAttribute` Base Class

All field attributes extend `FieldAttribute`:

```php
#[Attribute(Attribute::TARGET_PROPERTY)]
class FieldAttribute
{
    public function __construct(
        public string  $type = 'text',
        public ?string $label = null,
        public array   $rules = [],
        public array   $attributes = [],
        public array   $options = [],
        public mixed   $default = null,
        public ?string $coerce = null
    )
    {}
}
```

Key methods:

- `fromConfig(array $config): static` — Creates an attribute instance from a config array (used by `Form::add()`).
- `toFieldConfig(): array` — Extracts field config (label, rules, attributes, options, coerce) from the attribute for the `Field` object.

### Creating a Custom Field Attribute

```php
use Attribute;

#[Attribute(Attribute::TARGET_PROPERTY)]
class ColorField extends FieldAttribute
{
    public function __construct(
        ?string $label = null,
        array   $rules = [],
        array   $attributes = [],
        mixed   $default = '#000000'
    )
    {
        parent::__construct(
            type: 'color',
            label: $label,
            rules: $rules,
            attributes: $attributes,
            default: $default
        );
    }
}
```

Usage:

```php
#[ColorField(label: 'Background Color')]
protected string $bgColor;
```

## How `parseAttributes()` Works

This is the core engine that discovers fields from your form's typed properties:

```php
protected function parseAttributes(): void
{
    $reflection = new ReflectionClass($this);
    $properties = $reflection->getProperties(
        ReflectionProperty::IS_PUBLIC | ReflectionProperty::IS_PROTECTED
    );

    foreach ($properties as $property) {
        $attributes = $property->getAttributes(
            FieldAttribute::class,
            \ReflectionAttribute::IS_INSTANCEOF
        );

        foreach ($attributes as $attribute) {
            $instance = $attribute->newInstance();

            $defaultValue = $instance->default;
            if ($property->isInitialized($this) && $property->getValue($this) !== null) {
                $defaultValue = $property->getValue($this);
            }

            $this->add($property->getName(), $instance->type, [
                'label' => $instance->label,
                'rules' => $instance->rules,
                'attributes' => $instance->attributes,
                'options' => $instance->options,
                'value' => $defaultValue,
                'coerce' => $instance->coerce,
            ]);

            if ($defaultValue !== null && !$property->isReadOnly()) {
                $property->setAccessible(true);
                $property->setValue($this, $defaultValue);
            }
        }
    }
}
```

Step by step:
1. Uses reflection to find all public and protected properties on the form class
2. For each property, looks for attribute annotations that are instances of `FieldAttribute`
3. For each matching attribute, creates an instance and extracts its configuration
4. Calls `$this->add()` to create a `Field` object
5. Sets the form's property to the default value (if one exists)

## Hiding Attributes from Serialization

The `#[Hidden]` attribute marks model properties that should be excluded from `toArray()` and `toJson()` output. This is useful for sensitive data like passwords.

### Using `#[Hidden]`

```php
use Strux\Component\Model\Attributes\Hidden;

class User extends Model
{
    #[Column]
    #[Hidden]
    public string $password;

    #[Column]
    #[Hidden]
    public string $rememberToken;
}
```

### `hide()` and `unhide()`

Temporarily hide or reveal attributes on a model instance:

```php
$user = User::find(1);

// Hide additional fields
$user->hide(['email', 'phone'])->toArray();

// Reveal hidden fields
$user->unhide(['password'])->toArray();
```

### Conditional Visibility

Pass a closure to conditionally apply hide/unhide:

```php
// Hide email only for non-admin users
$user->hide(['email'], fn() => !$user->isAdmin);

// Reveal password only in debug mode
$user->unhide(['password'], fn() => config('app.debug'));
```

### Direct Access Still Works

Hiding only affects serialization. Direct property access always works:

```php
$user = User::find(1);
$user->toArray();  // password excluded
$user->password;   // still accessible directly
echo $user->password;  // works
```

### Collections

`hide()` and `unhide()` work on collections too:

```php
$users = User::all()->hide(['password', 'rememberToken']);
echo json_encode($users);  // password and token excluded from all users
```

## Forms in Controllers (Complete Example)

### Create Action

```php
#[Route('/create', methods: ['GET', 'POST'], name: 'admin.products.create')]
public function create(Request $request): Response
{
    $form = new ProductForm();

    if ($request->is('POST')) {
        $form->bind($request);

        if ($form->isValid()) {
            $product = new Product();
            $product->title = $form->get('title');
            $product->slug = $form->get('slug');
            $product->price = (float) $form->get('price');
            $product->stock = (int) $form->get('stock');

            if ($this->productRepository->save($product)) {
                $this->flash->set('success', 'Product created.');
                return $this->redirect(route('admin.products.index'));
            }
        }

        // Validation failed — fall through to re-render with errors
        return $this->view('admin/products/create', ['form' => $form], 400);
    }

    return $this->view('admin/products/create', ['form' => $form]);
}
```

### Edit Action

```php
#[Route('/edit/int:id', methods: ['GET', 'POST'], name: 'admin.products.edit')]
public function edit(Request $request, int $id): Response
{
    $product = $this->productRepository->findByIdWithCategories($id);

    if (!$product) {
        $this->flash->set('error', 'Product not found.');
        return $this->redirect(route('admin.products.index'));
    }

    // Auto-bind: the Product object populates the form fields
    $form = new ProductForm($product);

    if ($request->is('POST')) {
        $form->bind($request);  // Overwrite with submitted data

        if ($form->isValid()) {
            $this->mapFormDataToProduct($form, $product, $request);

            if ($this->productRepository->save($product)) {
                $this->flash->set('success', 'Product updated.');
                return $this->redirect(route('admin.products.index'));
            }
        }

        return $this->view('admin/products/edit', ['form' => $form, 'product' => $product], 400);
    }

    return $this->view('admin/products/edit', ['form' => $form, 'product' => $product]);
}
```

### Getting Values From the Form

```php
// Scalar values:

$title = $form->get('title', 'Default Title');    // with fallback

// Typed values:
$price = (float) $form->get('price');             // explicit cast
$stock = (int) $form->get('stock');               // explicit cast

// Arrays (checkboxes, multi-select):
$categories = $form->get('categories', []);       // falls back to empty array
if (!empty($categories) && is_array($categories)) {
    foreach ($categories as $catId) {
        $product->categories()->attach((int) $catId);
    }
}

// Via coercion (automatic):
$form->field('isFeatured')->getValue();  // Returns bool if BooleanField (coerce: 'bool')
```

### Re-populating on Error

After a POST fails validation, the form automatically retains the submitted values because `bind()` was already called:

```php
if ($request->is('POST')) {
    $form->bind($request);  // Values now contain user input

    if (!$form->isValid()) {
        // Render with user's input + error messages — no manual repopulation needed
        return $this->view('contact', ['form' => $form], 400);
    }
}
```

## Common Pitfalls

1. **Checkbox fields always submit a value** — The hidden+checkbox dual input means a BooleanField always submits either `0` or `1`. Use `BooleanField` (not `StringField`) for boolean toggles.

2. **SelectMultipleField and ListField names** — They render with `name="field[]"` so PHP receives an array. Always use `coerce: 'array'` (automatically applied on these types).

3. **Default values vs bound values** — The default value from the attribute is only used if no data is bound. Once `bind()` is called, the bound value overrides the default.

4. **Empty strings vs null** — The field value after binding an empty input is an empty string `""`, not `null`. The form's property setter converts `null` to `""` for `string`-typed properties.

5. **Date binding from models** — When binding a `DateTime` property from a model, the form converts it to `Y-m-d\TH:i:s` format. Your database stores dates as `Y-m-d H:i:s` — the form handles the conversion automatically.

6. **Protected properties** — Form field properties are `protected`. This is intentional: the `__get()` magic method intercepts access and returns the `Field` object instead of the raw string value. Direct property access (`$form->title`) returns the `Field` object; use `$form->get('title')` for the value.

7. **Validation order** — Rules are validated in the order they are defined. The first rule to fail stops further rule checks for that field (the `break` in `validateField()`).

8. **`render()` vs manual composition** — Use `render()` for simple forms (settings, quick CRUD). Use manual composition for complex forms with icons, custom layouts, or mixed field groupings.

9. **Grid layout with `render()`** — When using a grid class on the form element, each field wrapper becomes a grid cell. Use the layout callback to add `md:col-span-2` for fields that should span full width.

10. **`#[Hidden]` is serialization-only** — Hidden properties are still persisted to the database and accessible directly. They're only excluded from `toArray()` and `toJson()` output.
