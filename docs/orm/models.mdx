# ORM Models: The Heart of Your Application

A **Model** is a PHP class that represents a table in your database. Each Model instance represents a single row in that table. If you have a `users` table, you have a `User` model. If you have a `products` table, you have a `Product` model.

Strux uses the **Active Record** pattern. This means your data AND the logic to save/load that data live together in the same object.

> [!TIP]
> Think of a Model as a "smart container" for a database row. You fill it with data, tell it to save, and it handles all the SQL for you.

---

## 1. Creating a Model

### Generating a Model via CLI

The fastest way to create a model is using the `new:entity` command (also aliased as `new:model`, `g:e`, or `g:m`):

```bash
php bin/console new:entity User
```

This creates a `User` model at `src/Domain/General/Entity/User.php` with a basic skeleton:

```php
#[Entity(table: 'users')]
class User extends Model
{
    #[Id, Column(type: Field::integer)]
    public ?int $id = null;

    #[Column(type: Field::string, length: 150)]
    public string $name;
}
```

**Options:**

| Option | Short | Description |
|--------|-------|-------------|
| `--domain` | `-d` | Domain directory to place the entity in (default: `General`) |
| `--migrate` | `-m` | Also generate a migration file immediately after creating the entity |
| `--uuid` | | Use UUID as the primary key type instead of auto-increment integer |
| `--ulid` | | Use ULID as the primary key type instead of auto-increment integer |

```bash
php bin/console new:entity Product --domain=Catalog
php bin/console new:entity Product --domain=Catalog --migrate
php bin/console new:entity Product --uuid
php bin/console g:e Product           # short alias
php bin/console g:m Product           # short alias
```

> [!TIP]
> Passing `--migrate` runs `db:migrate` targeting the new model right away, so you can go from zero to a migrated table in one command.

### Creating a Model Manually

Every model must extend the abstract `Strux\Component\Database\ORM\Model` class.

```php
use Strux\Component\Database\Schema\Attributes\Entity;
use Strux\Component\Database\Schema\Attributes\Column;
use Strux\Component\Database\Schema\Attributes\Id;
use Strux\Component\Database\Schema\Types\Field;
use Strux\Component\Database\ORM\Model;

#[Entity(table: 'users')]
class User extends Model
{
    #[Id, Column(type: Field::integer)]
    public ?int $id = null;

    #[Column(type: Field::string, length: 150)]
    public string $email;

    #[Column(type: Field::string)]
    public string $password;

    #[Column(type: Field::boolean)]
    public bool $active = true;
}
```

> [!IMPORTANT]
> Properties with `#[Column]` MUST be **public**. Protected and private properties are ignored by the ORM. This is by design — only public properties are persisted to the database.

### What Happens When You Create a Model

1. The constructor is called with optional initial attributes.
2. The ORM resolves the database connection from the container.
3. It boots all traits (more on traits later).
4. It fills the model with any provided attributes.
5. It determines the table name from the `#[Entity]` attribute or by pluralizing the class name.
6. It discovers the primary key from the `#[Id]` attribute.

## 2. CRUD: Create, Read, Update, Delete

These are the four fundamental operations you can perform on any model.

### Creating Records

#### Method 1: Create and Save

```php
$user = new User();
$user->email = 'john@example.com';
$user->password = password_hash('secret123', PASSWORD_BCRYPT);
$user->active = true;

$result = $user->save(); // Returns true on success, false on failure

if ($result) {
    echo "User created with ID: " . $user->id;
}
```

#### Method 2: Static Create

```php
$user = User::create([
    'email' => 'john@example.com',
    'password' => password_hash('secret123', PASSWORD_BCRYPT),
    'active' => true,
]);
```

The `create()` method:
1. Creates a new instance of the model.
2. Fills it with the provided attributes.
3. Calls `save()`.
4. Returns the model instance with the ID populated.

#### Method 3: Construct with Attributes

```php
$user = new User([
    'email' => 'john@example.com',
    'password' => password_hash('secret123', PASSWORD_BCRYPT),
    'active' => true,
]);
$user->save();
```

> [!WARNING]
> `create()` and `fill()` ONLY set properties that have the `#[Column]` attribute. Non-column properties are ignored. This prevents accidentally saving internal state to the database.

### Reading Records

#### Find by Primary Key

```php
$user = User::find(1); // Returns the User with ID 1, or null if not found
```

You can eager-load relations directly with `find()`:

```php
$user = User::with('posts', 'profile')->find(1);
```

> `with()` is a convenient alias for `include()`. Both methods work identically.

#### Find or Fail

```php
$user = User::findOrFail(1); // Returns the User, or throws an exception
```

#### Get All Records

```php
$users = User::all(); // Returns a Collection of all users
```

#### Get the First Record

```php
$user = User::first(); // Returns the first user, or null
```

#### Get the Last Record

```php
$user = User::last(); // Returns the last user (ordered by primary key DESC), or null
```

#### Get with Conditions

```php
$activeUsers = User::where('active', 1)->get();
$firstActive = User::where('active', 1)->first();
```

> [!TIP]
> `User::all()` is equivalent to `User::query()->get()`. Both return a `Collection`.

### Updating Records

#### Method 1: Find, Modify, Save

```php
$user = User::find(1);
$user->email = 'newemail@example.com';
$user->save();
```

#### Method 2: Static Update

```php
$user = User::update(1, [
    'email' => 'newemail@example.com',
    'active' => false,
]);
```

> [!NOTE]
> `User::update()` returns the updated model, or `null` if the record was not found.

### Deleting Records

#### Delete a Single Model

```php
$user = User::find(1);
$user->delete(); // Returns true if deleted, false otherwise
```

#### Delete by ID (Static)

```php
User::destroy(1);        // Delete a single record
User::destroy([1, 2, 3]); // Delete multiple records
```

The `destroy()` method:
- Returns the number of deleted records.
- Works with both single IDs and arrays of IDs.
- For composite primary keys, pass arrays of key-value pairs.

## 3. How `save()` Works — The Full Journey

When you call `save()`, the ORM goes through a carefully orchestrated sequence of steps:

### For a New Record (INSERT)

```
1. beforeSave() hook fires
2. Validation runs (if rules are defined)
3. Saving event fires (cancelable)
4. Timestamps are handled (created_at / updated_at)
5. Creating event fires (cancelable)
6. Properties are prepared for the database:
   - Booleans become 1 or 0
   - Arrays become JSON strings
   - DateTime objects become formatted strings
7. Auto-generated IDs are created (UUID/ULID if configured)
8. SQL INSERT is executed
9. Last insert ID is captured
10. _exists flag is set to true
11. Original state is saved for dirty tracking
12. Created event fires
13. Saved event fires
14. afterCreate() hook fires
15. afterSave() hook fires
```

### For an Existing Record (UPDATE)

```
1. beforeSave() hook fires
2. Validation runs (if rules are defined)
3. Saving event fires (cancelable)
4. Timestamps are handled (updated_at)
5. Properties are compared to original state (dirty checking)
6. If nothing changed -> returns true immediately (no query)
7. Updating event fires (cancelable)
8. Only changed columns are included in the UPDATE
9. SQL UPDATE is executed
10. Original state is refreshed
11. Updated event fires
12. Saved event fires
13. afterUpdate() hook fires
14. afterSave() hook fires
```

> [!IMPORTANT]
> Step 6 is a major performance feature. If you call `$user->save()` without changing anything, NO database query is executed. This makes it safe to call `save()` in loops or after conditional modifications.

## 4. Dirty State Tracking

Your models "remember" their original state — the values they had when they were loaded from the database. This is called **Dirty Tracking**.

```php
$user = User::find(1);

// Nothing has changed yet
if ($user->isDirty()) {
    echo "This will NOT execute.";
}

$user->email = 'changed@example.com';

// Something changed!
if ($user->isDirty()) {
    echo "Something has been modified!";
}

// Check a specific property
if ($user->isDirty('email')) {
    echo "The email was changed!";
}

// Get the original value
$oldEmail = $user->getOriginal('email'); // Returns the value before modification

// Get all original values
$originals = $user->getOriginal(); // Returns array of all original values

// Get only the changes
$changes = $user->getChanges(); // Returns ['email' => 'changed@example.com']
```

### Practical Uses for Dirty Tracking

#### Conditional Logic

```php
$user = User::find(1);
$user->status = 'active';

if ($user->isDirty('status')) {
    $oldStatus = $user->getOriginal('status');
    sendStatusChangeEmail($user, $oldStatus, 'active');
}

$user->save();
```

#### Audit Logging

```php
foreach ($user->getChanges() as $field => $newValue) {
    $oldValue = $user->getOriginal($field);
    AuditLog::create([
        'model' => 'User',
        'model_id' => $user->id,
        'field' => $field,
        'old_value' => $oldValue,
        'new_value' => $newValue,
    ]);
}
```

## 5. Serialization: Converting Models to Arrays and JSON

Models can be easily converted to arrays or JSON for APIs and frontend consumption.

### toArray()

```php
$user = User::find(1);
$array = $user->toArray();
// Returns: ['id' => 1, 'email' => 'john@example.com', 'active' => true]
```

### toJson()

```php
$user = User::find(1);
$json = $user->toJson(); // Returns a JSON string
$prettyJson = $user->toJson(JSON_PRETTY_PRINT); // Formatted JSON
```

### Collection Serialization

```php
$users = User::all();
$array = $users->toArray(); // Array of arrays
$json = json_encode($users); // Works because Collection implements JsonSerializable
```

### Hiding Fields from Serialization

Sometimes you want to exclude certain fields from serialization (like passwords).

#### Using the `#[Hidden]` Attribute

```php
use Strux\Component\Database\ORM\Attributes\Hidden;

class User extends Model
{
    #[Column(type: Field::string)]
    #[Hidden] // This field will NOT appear in toArray() or toJson()
    public string $password;
}
```

#### Runtime Hiding

```php
$user = User::find(1);

// Hide fields dynamically
$user->hide(['password', 'secret_key']);

// Or hide conditionally
$user->hide(['email'], function () {
    return !currentUser()->isAdmin();
});

// Show hidden fields again
$user->unhide(['password']);

$array = $user->toArray(); // Hidden fields excluded
```

> [!TIP]
> `hide()` and `unhide()` also work on Collections: `$users->hide(['password'])` hides the field on every model in the collection.

## 6. The `getAttribute()` Method

Access individual attributes safely, falling back to the original state if the property is not publicly accessible:

```php
$user = User::find(1);

// Direct property access (fastest)
$email = $user->email;

// Via getAttribute() (works for original values too)
$email = $user->getAttribute('email');

// Get original state
$original = $user->getOriginal();
$originalEmail = $user->getOriginal('email');
```

## 7. Model State: `exists()` and `isNew()`

Check whether a model exists in the database:

```php
$user = new User();
$user->isNew();  // true - never been saved
$user->exists(); // false - not in the database

$user->save();

$user->isNew();  // false
$user->exists(); // true - now in the database

$savedUser = User::find(1);
$savedUser->isNew();  // false
$savedUser->exists(); // true
```

## 8. Table Name and Primary Key Resolution

### How the Table Name is Determined

```php
// Explicit table name via #[Entity]
#[Entity(table: 'custom_table_name')]
class User extends Model {}

// Or auto-generated from the class name
class BlogPost extends Model {}  // Table name: 'blog_posts'
```

The auto-generation rule:
1. Take the short class name (e.g., `BlogPost`).
2. Convert from PascalCase to snake_case: `blog_post`.
3. Add an "s" at the end: `blog_posts`.

> [!WARNING]
> Always check the auto-generated table name. Some irregular plurals might not match your expectations. When in doubt, specify the table name explicitly with `#[Entity(table: 'your_table')]`.

### How the Primary Key is Determined

The ORM scans all public properties for the `#[Id]` attribute:

```php
// Single primary key
#[Id, Column(type: Field::integer)]
public ?int $id = null;

// Multiple primary keys (composite key)
#[Id, Column(type: Field::integer)]
public ?int $orderId = null;

#[Id, Column(type: Field::integer)]
public ?int $productId = null;
```

If no property has `#[Id]`, the ORM assumes a column named `id`.

> [!TIP]
> You can check if a model uses composite keys: `$user->isCompositeKey()` returns `true` or `false`.

## 9. Timestamps

The `HasTimestamps` trait provides automatic `created_at` and `updated_at` handling.

### Enabling Timestamps

You need three things for timestamps to work:

1. The `#[Timestamps]` attribute on the class (for the migration engine).
2. Columns with `currentTimestamp` and/or `onUpdateCurrentTimestamp` (for the database schema).
3. The `HasTimestamps` trait is used by Model automatically (it is in the `use` statement of the Model class).

```php
use Strux\Component\Database\ORM\Behavior\HasTimestamps;
use Strux\Component\Database\Schema\Attributes\Column;
use Strux\Component\Database\Schema\Attributes\Timestamps;
use Strux\Component\Database\Schema\Types\Field;

#[Timestamps(enabled: true)]
class User extends Model
{
    // The 'createdAt' column
    #[Column(type: Field::datetime, currentTimestamp: true)]
    public ?string $createdAt = null;

    // The 'updatedAt' column
    #[Column(type: Field::datetime, onUpdateCurrentTimestamp: true)]
    public ?string $updatedAt = null;
}
```

### How Timestamps Work

1. When `save()` is called, `handleTimestamps()` is invoked.
2. For new records (`_exists === false`), it removes null `createdAt` values from the attributes so the database can use its default (`currentTimestamp`).
3. For all saves, it removes null `updatedAt` values so the database can update them via `onUpdateCurrentTimestamp`.

> [!NOTE]
> Timestamps are primarily handled at the **database level** using `CURRENT_TIMESTAMP` defaults. The ORM simply strips null values so the database defaults kick in. This means timestamps work correctly even if you bypass the ORM and insert data directly.

### Custom Timestamp Column Names

```php
#[Timestamps(
    createdAt: 'registered_at',
    updatedAt: 'modified_at'
)]
class User extends Model
{
    #[Column(type: Field::datetime, currentTimestamp: true)]
    public ?string $registeredAt = null;

    #[Column(type: Field::datetime, onUpdateCurrentTimestamp: true)]
    public ?string $modifiedAt = null;
}
```

### Checking if Timestamps are Enabled

```php
// These methods are available if you want to inspect the configuration
$hasTimestamps = $user->hasTimestamps();       // Returns true/false
$createdAtCol  = $user->getCreatedAtColumn();  // Returns column name or null
$updatedAtCol  = $user->getUpdatedAtColumn();  // Returns column name or null
```

## 10. Global Scopes

Global scopes add constraints to ALL queries on a model. They are like "always-on" filters.

### Adding a Global Scope

```php
use Strux\Component\Database\ORM\Model;

class User extends Model
{
    // This method is called during initialization
    public function initializeHasQueryBuilder(): void
    {
        static::addGlobalScope('active_users', function (Model $builder) {
            $builder->where('active', 1);
        });
    }
}
```

Now every query on the User model automatically includes `WHERE active = 1`:

```php
$users = User::all(); // SELECT * FROM users WHERE active = 1
```

### How Global Scopes are Applied

1. When you call `User::query()`, a new query builder instance is created.
2. During initialization, `applyGlobalScopes()` is called on the model instance.
3. All registered global scopes are applied to the query.
4. Every query — `get()`, `first()`, `count()`, `update()`, `delete()` — automatically includes the scope constraints.

### Removing a Global Scope

```php
// Remove for a single query
$users = User::query()->withoutGlobalScope('active_users')->get();

// Includes inactive users now
```

### The Soft Deletes Global Scope

The `HasSoftDeletes` trait automatically adds a global scope:

```php
// The SoftDelete trait adds this scope automatically:
protected function initializeHasSoftDeletes(): void
{
    static::addGlobalScope('softDeletes', function (Model $builder) {
        $column = $builder->getSoftDeleteColumn();
        $builder->where($column, 'IS NULL');
    });
}
```

This means all queries automatically exclude soft-deleted records.

> [!TIP]
> Global scopes are stored per model class. Different models can have different global scopes without interfering with each other.

## 11. Lazy Loading Prevention

By default, models will automatically lazy-load relationships when you access them as properties. You can prevent this and force explicit eager loading:

```php
// Globally prevent lazy loading (useful in development to catch N+1 queries)
User::preventLazyLoading(true);

// Now accessing an unloaded relationship throws a LazyLoadingViolationException
$user = User::find(1);
$roles = $user->roles; // Throws LazyLoadingViolationException!

// You must eager-load explicitly: use with() (or include())
$user = User::with('roles')->find(1);
$roles = $user->roles; // OK — already loaded
```

> [!TIP]
> Enable `preventLazyLoading()` in your development environment to catch N+1 query issues early. Disable it in production for the normal lazy loading behavior.

## 12. Model Lifecycle Summary

Here is the complete lifecycle of a model from creation to deletion:

```
  new Model()
      |
      v
  resolveConnection()  -> Get PDO from container
      |
      v
  bootTraits()         -> Run initialize[TraitName]() methods
      |
      v
  fill(attributes)     -> Set properties, apply transforms
      |
      v
  save()
      |
      +-- beforeSave() hook
      +-- validate() -> Runs #[Validate] rules
      +-- Saving event (cancelable)
      +-- handleTimestamps()
      |
      +-- [NEW] _performInsert()
      |     +-- Creating event (cancelable)
      |     +-- Generate UUID/ULID if needed
      |     +-- Execute INSERT
      |     +-- Capture lastInsertId
      |     +-- Set _exists = true
      |     +-- Created event
      |
      +-- [EXISTING] _performUpdate()
      |     +-- Check dirty state
      |     +-- If clean, skip (no query!)
      |     +-- Updating event (cancelable)
      |     +-- Execute UPDATE (only changed columns)
      |     +-- Updated event
      |
      +-- Save original state
      +-- Saved event
      +-- afterSave() hook
```

## 13. Complete Model Method Reference

### Static Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `query()` | `static::query(): Model` | Returns a new query builder instance |
| `create()` | `static::create(array $attributes): static` | Creates and saves a new model |
| `update()` | `static::update(mixed $id, array $attributes): ?static` | Finds and updates a record |
| `find()` | `static::find(mixed $id, array $includes = []): ?static` | Finds by primary key, with optional eager loading |
| `with()` | `static::with(mixed ...$relations): static` | Eager load relations (alias of `include()`) |
| `include()` | `static::include(mixed ...$relations): static` | Eager load relations |
| `findOrFail()` | `static::findOrFail(mixed $id): static` | Finds by PK, throws if not found |
| `all()` | `static::all(): Collection` | Returns all records |
| `destroy()` | `static::destroy(mixed $ids): int` | Deletes by primary key(s) |
| `transaction()` | `static::transaction(Closure $callback): mixed` | Runs a closure in a transaction |
| `beginTransaction()` | `static::beginTransaction(): void` | Starts a transaction |
| `commit()` | `static::commit(): void` | Commits a transaction |
| `rollBack()` | `static::rollBack(): void` | Rolls back a transaction |
| `preventLazyLoading()` | `static::preventLazyLoading(bool $prevent): void` | Enables/disables lazy loading |
| `disableEvents()` | `static::disableEvents(): EventDisabler` | Returns an event disabler configurator |
| `enableAllEvents()` | `static::enableAllEvents(): void` | Re-enables all events |
| `fromStorage()` | `static::fromStorage(array $data): static` | Creates a model from database data |

### Instance Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `save()` | `save(): bool` | Persists the model to the database |
| `delete()` | `delete(): bool` | Deletes the model from the database |
| `fill()` | `fill(array $attributes): static` | Sets attributes from an array |
| `exists()` | `exists(): bool` | Checks if the model exists in the database |
| `isNew()` | `isNew(): bool` | Checks if the model has never been saved |
| `getAttribute()` | `getAttribute(string $key): mixed` | Gets an attribute by name |

| `hide()` | `hide(array $fields, ?callable $condition): static` | Hides fields from serialization |

| `toArray()` | `toArray(): array` | Converts the model to an array |
| `toJson()` | `toJson(int $options): string` | Converts the model to JSON |
| `getTable()` | `getTable(): string` | Returns the table name |
| `getPrimaryKey()` | `getPrimaryKey(): string` | Returns the primary key name |
| `getPrimaryKeys()` | `getPrimaryKeys(): array` | Returns all primary key names |
| `isCompositeKey()` | `isCompositeKey(): bool` | Checks if the model uses composite keys |
| `getLastInsertId()` | `getLastInsertId(?string $name): string\|false` | Gets the last inserted ID |
| `removeOriginalKey()` | `removeOriginalKey(string $key): void` | Removes a key from original state |

### Internal Lifecycle Hooks (override in your model)

| Hook | Called During |
|------|---------------|
| `beforeSave()` | Before every save (create or update) |
| `afterSave()` | After every save (create or update) |
| `beforeCreate()` | Before a new record is inserted |
| `afterCreate()` | After a new record is inserted |
| `beforeUpdate()` | Before an existing record is updated |
| `afterUpdate()` | After an existing record is updated |
| `beforeDelete()` | Before a record is deleted |
| `afterDelete()` | After a record is deleted |
